## Main Direction

This area should likely stay split across multiple layers instead of forcing everything into one flat category:

- `Positional Escapes`
- `Submission Defense / Submission Escapes`
- `Leg Lock Escapes`
- `Shared Defense Families`

That matters because many entries are not really "moves" in the same sense as a sweep or pass. They are often recurring escape systems built around:

- frame
- posture
- hand fight
- hide limb
- clear line
- recover guard
- turn to knees
- stand up

## Shared Defense Families

This may end up being the most useful layer because it reveals recurring principles across multiple escapes.

### A. Elbow-Line Escapes

Used for:

- `Armbar`
- `Short Armbar`
- `Cutting Armbar`
- some `Kimura` finishes once elbow slips

Core idea:

- keep elbow from crossing control line
- connect hands
- pull elbow below fulcrum
- rotate body to free line

### B. Shoulder-Line Escapes

Used for:

- `Kimura`
- `Americana`
- `Omoplata`
- `Monoplata-Type Attacks`

Core idea:

- reduce shoulder rotation
- straighten arm if possible
- connect hands
- turn body toward safe angle
- roll only when it actually removes pressure

### C. Posture-And-Angle Escapes

Used for:

- `Triangle`
- `Omoplata`
- `Loop Choke`
- collar attacks from guard
- `Guillotine` entries before lock is fully set

Core idea:

- recover posture
- prevent opponent angle
- hide vulnerable shoulder / neck line
- break lock before final bite

### D. Hand-Fight-First Choke Escapes

Used for:

- `Rear Naked Choke`
- collar chokes
- `Guillotine`
- `D'Arce`
- `Anaconda`
- `Ezekiel`

Core idea:

- first address choking structure, not random movement
- peel top hand or finishing grip
- rebuild neck posture
- turn toward safest exit

### E. Knee-Line Escapes

Used for:

- `Heel Hook`
- `Ankle Lock`
- `Toe Hold`
- `Kneebar`
- `Aoki Lock`
- `Estima Lock`

Core idea:

- if knee line is trapped, danger remains
- hide heel or foot
- hand fight
- free secondary leg
- clear knee line before anything else

### F. Frame-Shrimp-Reguard Escapes

Used for:

- `Mount`
- `Side Control`
- `North-South`
- `Knee-On-Belly`
- some `Body Lock` situations

Core idea:

- build frames
- move hips, not just push
- recover knee inside
- regain guard or half guard

### G. Bridge-Turn-Knees Escapes

Used for:

- `Side Control`
- turtle pressure
- `Mount`
- `Scarf Hold`
- `Front Headlock` scenarios

Core idea:

- off-balance opponent
- create turn angle
- come to knees or single-leg style recovery

### H. Hook-Removal Escapes

Used for:

- `Back Control`
- `Truck`
- `Twister` setups
- some `Crab Ride` situations

Core idea:

- control choking side first
- remove lower hook or trapping leg
- put back / shoulder to mat
- turn into opponent

## Important Modeling Note

A lot of these are not "moves" in the same way a sweep is a move.

They are often better modeled as:

- escape systems
- defense families
- recurring principles
- or grouped branches under a larger attack family

So when this is added into the Index, it should likely not become one giant flat list of every named escape variation.

Safer approach:

1. Add the family hubs first
2. Add the highest-value named escapes beneath them
3. Cross-reference them from the matching submissions / positions
4. Use the shared defense families as the conceptual glue
